The proposer is queried from each CL node's /status endpoint. Each node reports its own address and the current proposer address. The first non-zero proposer address is resolved to a node name. The proposer node is highlighted with a star on the graph and a purple name in the heights list. Updated on every topology poll.
Shows two vertical bars to the left of each node representing EL transaction pool fill levels.
■ Left bar = queued transactions (orange)
■ Right bar = pending transactions (yellow)
Each square = 1000 txs. Max squares derived from manifest txpool config (default 10000 txs = 10 squares).
Data from txpool_status RPC on each EL node. Polling starts when this toggle is enabled.
Shows one-way network latency (ms) on each graph edge based on the AWS region assigned to each node.
Values come from the AWS inter-region latency matrix (cloudping.co P50 medians, RTT/2).
Nodes in the same region show 0ms. Only visible when latency emulation is enabled in the manifest.
Draws dashed shapes around nodes that belong to the same subnet.
Subnets define network isolation boundaries. Nodes on different subnets can only communicate through bridge nodes present on both subnets.
Only available in Graph view.
Overlays CL persistent peer connections on the graph with directional arrows.
A→B means A has B configured as a persistent peer in the manifest.
Persistent peers use direct gossipsub delivery, bypassing mesh routing.
Data from the manifest's cl_persistent_peers field.
Overlays EL trusted peer connections on the graph with directional arrows.
A→B means A has B configured as a trusted peer in the manifest.
Trusted peers bypass the default peer scoring and are always maintained.
Data from the manifest's el_trusted_peers field.
